# Copilot Governance Instructions for CNF Token Project

## Core Principles
- **Zero Trust Architecture**: All changes must be validated through guard systems
- **Policy-Driven Development**: All operations must comply with `policy/rules.yaml`
- **Immutable Core**: Files in `/core` directory are strictly protected
- **Audit Trail**: All operations must be logged via `guard.audit_logger`
- **Test-First Approach**: Tests must be written before implementation

## Guard System Integration

### Core Protection (`guard.core_integrity`)
- **FORBIDDEN**: Direct modification of files in `/core` directory
- **REQUIRED**: Use `guard.core_integrity` for any core-related operations
- **VALIDATION**: All core changes must pass integrity hash validation
- **AUDIT**: Core modifications are logged as CRITICAL events

### Change Validation (`guard.change_validator`)
- **REQUIRED**: Analyze all changes before committing
- **ENFORCEMENT**: Prevent unauthorized file modifications
- **CATEGORIZATION**: Changes must be classified (staged/unstaged/untracked)
- **SAFETY**: Validate change safety before proceeding

### Token Drift Detection (`guard.token_drift`)
- **MONITORING**: Track token consistency across operations
- **THRESHOLDS**: Respect drift level thresholds (normal/warning/critical)
- **TRENDING**: Monitor drift trends (upward/downward/stable)
- **ADAPTIVE**: Use adaptive threshold calculation

### Auto Staging (`guard.auto_stage`)
- **LEVELS**: Respect staging levels (conservative/normal/aggressive)
- **PROTECTION**: Never auto-stage protected paths (core/, .git/, secrets/)
- **CATEGORIZATION**: Files must be categorized before staging
- **VALIDATION**: Dry-run staging before actual execution

### Commit Control (`guard.commit_control`)
- **ATOMIC**: Enforce atomic commits (all related changes together)
- **TYPES**: Use proper commit types (feat/fix/test/chore/docs/style/refactor/perf)
- **VALIDATION**: Validate commit messages and scope
- **BREAKING**: Track breaking changes appropriately

### Dual Execution (`guard.dual_execution`)
- **CONSISTENCY**: Validate dual execution results
- **LEVELS**: Respect consistency levels (strict/semantic/permissive)
- **PERFORMANCE**: Monitor execution time variance (≤50% allowed)
- **COMPARISON**: Compare semantic equivalence of results

### Audit Logging (`guard.audit_logger`)
- **MANDATORY**: All operations must be logged
- **LEVELS**: Use appropriate log levels (INFO/WARNING/ERROR/CRITICAL)
- **CONTEXT**: Include operation context and metadata
- **SEARCH**: Support log searching and filtering

### Enforcement Rules (`guard.enforcement_rules`)
- **REGISTRATION**: Register rules with proper metadata
- **DEPENDENCIES**: Respect rule dependencies
- **EXECUTION**: Execute rules with performance tracking
- **AUTO-FIX**: Support automatic rule fixes where safe

### State Management (`guard.state_manager`)
- **CHECKPOINTS**: Create checkpoints before risky operations
- **VALIDATION**: Validate state before operations
- **ROLLBACK**: Support rollback to previous states
- **COMPARISON**: Compare checkpoint differences

### Security Validation (`guard.security_validator`)
- **SCANNING**: Perform security scans on files
- **PARALLEL**: Use parallel scanning for efficiency
- **THREATS**: Detect and report security threats
- **QUARANTINE**: Isolate suspicious files

### Test Validation (`guard.test_validator`)
- **EXECUTION**: Run tests automatically
- **PARALLEL**: Execute tests in parallel
- **RETRY**: Implement retry logic for flaky tests
- **REPORTING**: Generate detailed test reports

### Performance Guard (`guard.performance_guard`)
- **MONITORING**: Track performance metrics
- **THRESHOLDS**: Respect performance thresholds
- **OPTIMIZATION**: Suggest performance improvements
- **ALERTING**: Alert on performance degradation

### Guard Status (`guard.guard_status`)
- **DASHBOARD**: Provide system status overview
- **HEALTH**: Monitor guard system health
- **METRICS**: Track guard performance metrics
- **ALERTS**: Report guard system issues

## Policy Engine Integration

### Policy Enforcement (`policy.engine`)
- **MANDATORY**: All operations must comply with active policies
- **VALIDATION**: Validate policy configuration before use
- **ENFORCEMENT**: Execute policies with detailed reporting
- **VIOLATIONS**: Report policy violations with severity levels

### Policy Rules (`policy/rules.yaml`)
- **CORE_IMMUTABLE**: Prevent core directory modifications
- **GUARD_SYSTEM**: Ensure all required guards are operational
- **COMMIT_CONTROL**: Enforce commit policies
- **AUTO_STAGE**: Control automatic staging behavior
- **CI**: Validate CI/CD pipeline requirements
- **README_LAW**: Enforce documentation requirements

### Policy Types
- **CORE_PROTECTION**: Protecting core system integrity
- **CODE_QUALITY**: Maintaining code quality standards
- **SECURITY**: Ensuring security compliance
- **GOVERNANCE**: Managing project governance
- **PERFORMANCE**: Monitoring performance requirements
- **COMPLIANCE**: Ensuring regulatory compliance

### Policy Severity Levels
- **INFO**: Informational policy notes
- **WARNING**: Non-blocking policy warnings
- **ERROR**: Blocking policy errors
- **CRITICAL**: Critical policy violations requiring immediate attention

## Development Workflow

### Pre-Implementation
1. **Policy Check**: Verify compliance with `policy/rules.yaml`
2. **Guard Validation**: Ensure all relevant guards are operational
3. **Audit Setup**: Initialize audit logging for the operation
4. **State Checkpoint**: Create state checkpoint if needed

### Implementation
1. **Guard Integration**: Use appropriate guard modules throughout
2. **Policy Compliance**: Ensure all actions comply with active policies
3. **Audit Logging**: Log all significant operations
4. **Error Handling**: Use `guard.failure` for error management

### Post-Implementation
1. **Test Execution**: Run comprehensive test suite
2. **Guard Validation**: Verify guards still operational
3. **Policy Compliance**: Confirm policy compliance maintained
4. **Audit Review**: Review audit logs for anomalies

## Quick Reference

### Essential Commands
```bash
# Validate all policies
python3 -c "from policy.engine import get_engine; print(get_engine().enforce_all())"

# Check guard status
python3 -c "from guard.guard_status import get_status; print(get_status().get_report())"

# Run core integrity check
python3 -c "from guard.core_integrity import validate_core_hash; validate_core_hash()"

# Execute test suite
python3 -m pytest tests/ -v
```
