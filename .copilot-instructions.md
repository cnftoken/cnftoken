## Scope

This protocol applies STRICTLY to:
- core CNF system
- compression engine
- guard system
- policy engine

This protocol is RELAXED for:
- utility scripts
- tests
- prototypes

Copilot must adjust strictness accordingly.

## Standard Interfaces (MANDATORY)

Copilot must use these interfaces:

policy_engine.validate(context)

guard.pre_check(context)
guard.inline_check(result)
guard.post_check(result)

risk_engine.aggregate(guard_outputs)

logger.audit(event)

## Risk Score Aggregation

risk_score = weighted_sum(guards)

Default weights:
- core_integrity: 0.3
- security: 0.25
- change_validation: 0.2
- compression: 0.15
- performance: 0.1

All weights must sum to 1.0

## Operation Modes

SYSTEM_MODES:

1. STRICT (production)
   - full guard enforcement
   - strict policy
   - no shortcuts

2. BALANCED (default)
   - full guard + optimized performance

3. EXPERIMENTAL
   - relaxed guard
   - higher compression allowed
   - must log everything

Copilot must detect context and adjust behavior.

## Performance Budget

Guard + Policy overhead must not exceed:

- 15% in BALANCED mode
- 25% in STRICT mode

If exceeded:
→ reduce guard frequency
→ batch validation
→ optimize checks


## Anti-Overengineering Rule

Copilot must:

- avoid unnecessary abstraction
- prefer simple solutions when safe
- not introduce new layers without justification

Complexity must be justified by measurable benefit.
